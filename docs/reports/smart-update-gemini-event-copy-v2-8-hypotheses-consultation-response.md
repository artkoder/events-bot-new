### 3. Where regression risk still exists

Самый большой риск сейчас — **пересушить факты при откате (over-correction)**.
Если мы просто скажем "не пиши метатекст" и "извлекай плотную суть", есть риск, что Gemma начнёт выдавать слишком атомарные, лишённые контекста обрывки.
Например, вместо плотного факта:
* `История любви Муслима Магомаева и Тамары Синявской`
Она может выдать:
* `Любовь` или `Магомаев и Синявская`.
Это разрушит Generation. Откат должен чётко требовать самодостаточных (self-contained), но компактных смыслов.

### 4. What we are underestimating

Мы всё ещё можем недооценивать **"въедливость" корня `посвящ*`** в весах модели, когда дело касается форматов "концерт", "выставка" или "лекция". Да, мы делаем Revise жёстче, но если текст изначально строится вокруг этого слова, удаление предложения на этапе Revise может оставлять логические дыры в тексте, которые модель не сможет бесшовно залатать. Generation prompt должен быть предельно ясен насчёт альтернатив.

### 5. Prompt guidance for Gemma

Вот конкретные блоки, учитывающие наши выводы:

**A. Extraction Prompt (вместо блока про трансформацию и agenda-safe)**
```text
- Извлекай чистые, плотные и самодостаточные факты (data points). 
- Не используй пустые обёртки вроде «Выставка носит название...», «Автор выставки — ...» или «Мероприятие пройдет...».
- Суть должна быть выражена ясно, но без потери контекста. Например, вместо «Тема: искусство» пиши «Жизнь и творчество художниц», а не просто одно слово.
- Если в источнике есть намерения («на встрече расскажут о...», «лекция познакомит с...»), извлекай только сам предмет обсуждения (например: «устройство платформы и её возможности»), отбрасывая канцелярскую рамку.
- Критическое требование: не используй слово/корень «посвящ». Заменяй его прямым фактом о предмете события.
```

**B. `_pre_extract_issue_hints` (очищенные от шаблонов предложений)**
```python
if any("посвящ" in (fact or "").casefold() for fact in raw_facts):
    hints.append("Критическое требование: не используй слово/корень «посвящ». Извлекай только саму суть события, без этого слова.")

if any(_INTENT_FACT_RE.match(fact or "") for fact in raw_facts):
    hints.append("В raw_facts есть слова-намерения («расскажут о...»). Извлеки только сам предмет или тему напрямую, удалив канцелярскую рамку.")

if any(_LABEL_STYLE_FACT_RE.match(fact or "") for fact in raw_facts):
    hints.append("Запрещены ярлыки вроде «Тема: ...» или «Идея: ...». Извлекай тему как самодостаточный факт без префиксов.")
```

**C. Standard Generation Prompt (`quality_block`)**
```text
- Запрещён бюрократический метатекст: «Мероприятие анонсирует...», «Проект призван стать...», «Будет представлен обзор...», «На мероприятии расскажут о...», «Презентация посвящена...».
- Разрешён естественный event framing: «Лекция о ...», «Спектакль исследует ...», «В центре внимания — ...».
- Важно: избегай overuse (злоупотребления) шаблонами. Не начинай предложения с одной и той же вводной конструкции (например, «В центре...») чаще одного раза на весь текст. Описывай суть напрямую.
```

**D. Revise Prompt (Policy Issues для посвящ*)**
```python
if item == "посвящ*":
    issues.append("КРИТИЧЕСКИЙ СБОЙ: найдено запрещённое слово «посвящ». Тебе ЗАПРЕЩЕНО делать косметическую замену. Полностью УДАЛИ предложение с этим словом и напиши суть иначе, используя естественный фрейминг («Лекция о ...», «Спектакль исследует ...» или «Главная тема — ...»).")
```
