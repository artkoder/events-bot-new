## 7. Findings

- Переносится не event-specific wording из `2673`, а сама идея разнесения fact extraction на разные micro-contracts.
- Для `lecture_person` самый полезный разрез оказался `cluster -> theme -> profiles`, а не `subject -> agenda -> program`.
- Это и есть путь к универсализации: единый большой prompt плохо переносится между shape-ами, а shape-aware atomic contracts переносятся лучше.
- Правило полного покрытия фактов лучше измеряется через explicit semantic slots, привязанные к исходному floor, чем через один exact-match список noun phrases.
