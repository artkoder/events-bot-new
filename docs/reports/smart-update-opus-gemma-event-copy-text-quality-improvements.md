## A. Проблемы текущего текста (по dry-run)

Прежде чем предлагать решения — вот конкретные quality defects, наблюдаемые в реальных outputs.

### A.1. Embellishment / unsupported prose

| Event | Пример из output | Проблема |
|-------|----------------|----------|
| 2660 | «результат её многолетних поисков и экспериментов» | Откуда «многолетних»? Нет в фактах. |
| 2660 | «размышления о хрупкости и прочности, о границе между реальностью и иллюзией» | Красиво, но ungrounded — в фактах «дуальность», а не «реальность и иллюзия» |
| 2745 | «Спектакль обещает стать глубоким и эмоциональным опытом» | Promo tone, не факт |
| 2745 | «заставляющим задуматься о ценности родственных связей» | Press-release filler |
| 2687 | «позволит глубже погрузиться в тему» | Generic filler, удалить без потери смысла |

**Паттерн**: Gemma, когда фактов мало или они абстрактны, заполняет бюджет embellishment. Текст становится длиннее, но не информативнее.

### A.2. Monotonous sentence structure

| Event | Пример | Проблема |
|-------|--------|----------|
| 2745 | «Спектакль рассказывает... Постановка исследует... Спектакль затрагивает... Постановка предлагает...» | 4 предложения подряд с одной структурой «Существительное + глагол» |
| 2687 | «Лекция расскажет... Лекция охватывает... Лекция расскажет...» | То же + «лекция» 6 раз в тексте |
| 2660 | «Художница исследует... Работы художницы — это размышления...» | Passive explanatory tone |

**Паттерн**: текст читается как учебник или пресс-релиз, не как журналистский анонс. Нет ритмического разнообразия.

### A.3. Generic headings

| Event | Heading | Проблема |
|-------|---------|----------|
| 2660 | «Основная идея выставки» | Можно поставить к любой выставке |
| 2660 | «Первый проект» | Неинформативный |
| 2687 | «Формат мероприятия» | Мета-зашумление |
| 2745 | «Ключевые мотивы постановки» | Шаблонный |
| 2660 exp | «О событии» | Самый generic heading |

### A.4. Weak leads

| Event | Lead | Проблема |
|-------|------|----------|
| 2660 | «Это дебютная выставка Анны Полтавец, на которой представлены работы...» | Formal, не цепляет |
| 2745 | «Спектакль «Сёстры» рассказывает о взаимоотношениях двух сестёр» | Прямолинейно, нет «крючка» |
| 2687 | «Лекция «Художницы» расскажет о вкладе...» | Метатекст: «расскажет о» вместо самого содержания |

**Паттерн**: почти все leads начинаются с «Это...» / «Спектакль рассказывает...» / «Лекция расскажет...». Монотонная метаструктура вместо содержательного входа.

## B. Prompt-level improvements (конкретные формулировки)

### B.1. Anti-embellishment rule

**Текущее (implicitly)**: нет — prompt говорит «не добавляй новых смыслов», но этого недостаточно.

**Предлагаемое — добавить в SHARED_DESCRIPTION_RULES:**

```
Grounding rule:
- Каждое утверждение в тексте должно прослеживаться к конкретному факту.
- НЕ добавляй: «обещает стать...», «позволит глубже погрузиться...», «заставляет задуматься о...», «представляет уникальный авторский стиль...».
- Если хочешь дополнить факт контекстом — используй ТОЛЬКО слова/образы из самого факта, не дорисовывай новые.
- Правило: можно перестроить фразу, нельзя достроить смысл.
```

**Expected impact**: устраняет 80% embellishment defects из A.1. Gemma хорошо следует explicit bans.

### B.2. Sentence variety rule

**Текущее**: нет.

**Предлагаемое — добавить в generation prompt:**

```
Ритм текста:
- Чередуй длинные и короткие предложения.
- Не начинай 2 предложения подряд с одного и того же слова (особенно «Спектакль...», «Лекция...», «Концерт...»).
- Используй разные конструкции: простое предложение → сложноподчинённое → назывное → вопросительное.
- Первое предложение секции — самое сильное. Не начинай секцию с «Также...» или «Кроме того...».
```

**Expected impact**: ломает монотонную структуру «Подлежащее + глагол» × 4 в ряд. Gemma 27b способна следовать таким style rules.

### B.3. Lead engineering

**Текущее (Style C)**: «Начни с короткой «картинки/настроения» из фактов.» — слишком размытая инструкция, Gemma часто игнорирует.

**Предлагаемое — заменить на явные lead patterns:**

```
LEAD RULES (выбери подходящий):

Pattern A — «Конкретная деталь»:
  Начни с самого яркого / необычного факта. Не с определения события.
  ✅ «Шесть художниц, каждая — со своим голосом: от барельефов Поленовой до беспредметных полотен Поповой.»
  ❌ «Лекция расскажет о вкладе художниц в историю русского искусства.»

Pattern B — «Вопрос/провокация» (если факты позволяют):
  ✅ «Что объединяет Серебрякову и Гончарову — кроме пола?»
  ❌ «Спектакль рассказывает о взаимоотношениях двух сестёр.»

Pattern C — «Цитата/голос» (если есть epigraph_fact):
  ✅ `> «В каждой работе застыла борьба мягкого и жёсткого»`
  ❌ Plain prose opening when a strong quote is available.

Pattern D — «Контраст»:
  ✅ «Барельеф — техника на стыке живописи и скульптуры: объём, который можно читать взглядом и трогать рукой.»
  ❌ «Художница работает в технике барельефа.»

ЗАПРЕТ: не начинай с метатекста «Это...» / «Данное мероприятие...» / «Лекция расскажет о...» / «Спектакль рассказывает...».
Вместо «лекция расскажет о Х» → пиши сразу про Х.
```

**Expected impact**: это самое высокоэффективное improvement. Меняет lead c «Wikipedia intro» на журналистский hook. Примеры в prompt работают как few-shot guidance для Gemma.

### B.4. Anti-metatext rule

**Текущее**: нет.

**Предлагаемое:**

```
Избегай метатекста:
- НЕ пиши «лекция расскажет о...» → пиши сразу содержание.
- НЕ пиши «спектакль рассказывает о...» → покажи что происходит.
- НЕ пиши «формат мероприятия — лекция» → читатель видит тип в карточке.
- НЕ пиши «выставка является...» / «концерт представляет собой...» → опиши что в этом особенного.

Принцип: описывай содержание, а не формат. Читатель уже знает, что это лекция/концерт/спектакль — из заголовка и карточки.
```

**Expected impact**: устраняет A.4 (weak leads) и значительную часть A.2 (monotonous structure). Метатекст — главный источник «учебникового» тона.

### B.5. Heading palette

**Текущее**: «избегай общих вроде «Подробности»» — один ban, недостаточно.

**Предлагаемое — расширенный ban list + positive guidance:**

```
Запрещённые headings:
«О событии», «О лекции», «О концерте», «О спектакле», «О выставке»,
«Подробности», «Основная идея», «Ключевые мотивы», «Формат мероприятия»,
«Описание», «Детали».

Хорошие headings — конкретные, с содержанием:
✅ «Шесть художниц» (вместо «О лекции»)
✅ «Репертуар» (вместо «Подробности»)
✅ «Барельеф как язык» (вместо «Основная идея»)
✅ «Что взять с собой» (вместо «Условия участия»)
✅ «На сцене» (вместо «Формат мероприятия»)
```

**Expected impact**: headings сами становятся content, а не markers. Telegraph страницы выглядят профессиональнее.

### B.6. Anti-duplication rule (prompt + runtime)

**Предлагаемое — в prompt:**

```
Каждый факт упоминается в тексте ровно ОДИН раз.
Не повторяй одну деталь в разных секциях.
Если факт уже в lead — не дублируй в body.
Если факт уже в body — не дублируй в другой секции.
```

**Предлагаемое — в runtime (после revise):**

```python
def _deduplicate_description_sentences(text: str) -> str:
    """Remove near-duplicate sentences from description."""
    sentences = _split_into_sentences(text)
    kept = []
    for s in sentences:
        s_words = set(re.findall(r'[а-яёa-z]{4,}', s.lower()))
        is_dup = False
        for prev in kept:
            prev_words = set(re.findall(r'[а-яёa-z]{4,}', prev.lower()))
            if len(s_words) > 3 and len(s_words & prev_words) / max(len(s_words), 1) > 0.6:
                is_dup = True
                break
        if not is_dup:
            kept.append(s)
    return ' '.join(kept)  # simplified; real impl would preserve paragraph structure
```

### B.7. Filler phrase ban list

**Текущее**: нет.

**Предлагаемое — добавить в запреты:**

```
Запрещённые филлеры (удалить эти обороты, текст не потеряет смысл):
- «обещает стать...»
- «позволит глубже погрузиться...»
- «заставит задуматься о...»
- «представляет уникальный...»
- «демонстрирует результат...»
- «является примером...»
- «даёт возможность...»
- «сделает вечер незабываемым...»
- «подарит массу впечатлений/эмоций...»
- «не оставит равнодушным...»
```

**Expected impact**: прямой ban list работает на Gemma лучше, чем абстрактное «без прессы». Gemma хорошо фильтрует по спискам.

## C. Engineering / architectural improvements для качества прозы

### C.1. Density-aware prompt selection

**Текущее**: один prompt для всех events.

**Предлагаемое**: 3 prompt variants по density:

| Density | Facts | Prompt variant | Key difference |
|---------|-------|---------------|----------------|
| Sparse (≤5) | Short facts | `compact_prompt` | «Напиши 2-4 предложения prose без headings. Не растягивай.» |
| Normal (6-12) | Medium | `standard_prompt` (current + improvements) | Lead + 2-3 ### |
| Dense (13+) | Many facts | `dense_prompt` | «Все факты обязательны. Используй списки для однородных items. 2-3 ### секции.» |

**Expected impact**: sparse events перестают быть over-inflated (2660, 2745). Dense events получают structural guidance для grouping.

### C.2. Lead variety enforcement (runtime)

```python
# After generation, check lead pattern
def _classify_lead(description: str) -> str:
    lead = description.split('###')[0].strip()
    if lead.startswith('>'):
        return 'quote_led'
    # Metatext patterns
    if re.match(r'(?i)(Это|Лекция|Спектакль|Концерт|Выставка)\s+(рассказыва|расскаж|представля|явля)', lead):
        return 'metatext_led'  # BAD
    if re.search(r'[?]', lead):
        return 'question_led'
    return 'content_led'  # default OK

lead_type = _classify_lead(description)
if lead_type == 'metatext_led':
    # Add to policy_issues for revise
    policy_issues.append(
        'Лид начинается с метатекста («Лекция расскажет...»). '
        'Перепиши: опиши содержание напрямую, без «расскажет о...».'
    )
```

**Expected impact**: catches metatext leads before they reach Telegraph. Revise prompt fixes them.

### C.3. Paragraph quality check

```python
def _has_filler_paragraph(text: str) -> list[str]:
    """Detect paragraphs that are pure filler (can be deleted without information loss)."""
    issues = []
    paragraphs = [p.strip() for p in re.split(r'\n{2,}', text) if p.strip() and not p.strip().startswith('###')]
    for p in paragraphs:
        # Short paragraph with no concrete nouns/numbers
        if len(p) < 120:
            has_concrete = bool(re.search(r'\d|[А-ЯЁ][а-яё]{2,}', p))  # number or proper noun
            if not has_concrete and any(f in p.lower() for f in [
                'позволит', 'обещает', 'заставит', 'подарит', 'даст возможность',
                'не оставит равнодушным', 'станет незабываемым'
            ]):
                issues.append(f'Абзац «{p[:60]}...» — филлер без конкретики, удали или замени фактом.')
    return issues
```

### C.4. Few-shot examples в prompt

**Текущее**: нет примеров хорошего текста в generation prompt. Gemma не видит образец.

**Предлагаемое**: добавить 1-2 short примера прямо в prompt:

```
ПРИМЕР ХОРОШЕГО ТЕКСТА (ориентир стиля, не копируй дословно):

> «В каждой работе застыла борьба мягкого и жёсткого»

Барельефы Анны Полтавец — на стыке живописи и скульптуры. Объём, который можно читать взглядом.

### Техника барельефа
Полтавец работает с рельефом: её фигуры выступают из плоскости, создавая диалог между поверхностью
и пространством. Это её первая персональная выставка.
```

**Expected impact**: few-shot examples — самый эффективный способ задать тон для Gemma. Один хороший пример стоит 10 rules. Budget ~100-150 tokens.

**Risk**: пример может «застрять» и Gemma будет копировать его structure для всех events. Mitigation: 2-3 разных примера, ротация.

### C.5. Post-generation sentence-level quality gate

Не только coverage check, но и prose quality check на level individual sentences:

```python
def _prose_quality_issues(text: str) -> list[str]:
    issues = []
    sentences = _split_into_sentences(text)

    # Check for consecutive same-start
    for i in range(1, len(sentences)):
        word_curr = sentences[i].split()[0] if sentences[i].split() else ''
        word_prev = sentences[i-1].split()[0] if sentences[i-1].split() else ''
        if word_curr == word_prev and len(word_curr) > 3:
            issues.append(
                f'Предложения {i} и {i+1} начинаются с одного слова «{word_curr}». '
                'Перестрой одно из них.'
            )

    # Check for metatext in lead
    lead_text = text.split('###')[0].strip().lstrip('> ').strip()
    if re.match(r'(?i)(Это|Лекция|Спектакль|Концерт|Выставка)\s+(рассказыва|расскаж|представля|явля|это\s)', lead_text):
        issues.append('Лид начинается с метатекста. Опиши содержание напрямую.')

    return issues
```

Эти issues идут в `_collect_policy_issues` → revise prompt их фиксит.

## E. Конкретный план: что добавить в prompt прямо сейчас

Если взять **один prompt change**, который даст максимальный quality lift, это:

### Composite Quality Block (вставить перед блоком «Запреты»)

```
Стиль текста:
1. Lead — без метатекста. НЕ начинай с «Лекция расскажет о...», «Спектакль рассказывает...», «Это...».
   Вместо этого → начни с самого яркого факта или конкретной детали напрямую.
2. Каждый факт упоминается ОДИН раз. Не повторяй деталь в разных секциях.
3. Не начинай 2 предложения подряд с одного слова.
4. Не пиши то, что можно удалить без потери информации: «обещает стать...», «позволит погрузиться...»,
   «заставит задуматься...», «не оставит равнодушным...» — это филлеры, удали.
5. Headings — конкретные и содержательные. Запрещены: «О событии», «Подробности», «Основная идея»,
   «Формат мероприятия», «Ключевые мотивы».
6. Принцип: описывай содержание, а не формат. Читатель уже знает тип события из карточки.
```
