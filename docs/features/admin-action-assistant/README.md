# Admin Action Assistant (Natural‑Language Commands)

## Зачем

В боте много команд и параметров, которые сложно помнить. Эта фича добавляет
**admin‑ассистента**: админ пишет запрос простыми словами, LLM (Gemma)
подбирает подходящее действие из allowlist, бот показывает **человекочитаемое
описание + команду**, и выполняет действие **только после подтверждения**.

## Входные точки

- `/assist (/a) <описание>` — однострочный запрос.
- `/assist` без аргументов — бот попросит описать действие следующим сообщением.
- Кнопка в главном меню (только для админов): `🧠 Описать действие`.

> Примечание: Telegram не поддерживает команды с кириллицей (вида `/описать`), поэтому
> «русский вход» реализован через кнопку меню и русскоязычные тексты.

## Примеры запросов (RU)

Чёткие запросы (ожидается сразу **proposals**):

- «Покажи команды» → `/help`
- «Открой меню» → `/menu`
- «Покажи события на 2026-02-24» → `/events 2026-02-24`
- «Общая статистика по авторазбору VK/TG и Gemma» → `/general_stats`
- «Покажи свежие импортированные события» → `/recent_imports`
- «Поставь таймзону +02:00» → `/tz +02:00`
- «Открой Telegram Monitoring» → `/tg`

Нечёткие запросы (ожидается **clarify** или несколько вариантов):

- «Покажи статистику» (может быть `/stats` или `/general_stats`)
- «Список событий за сутки» (может быть `/events`, `/recent_imports` или `/general_stats`, если неясно: нужен список событий или агрегаты)
- «Покажи очередь» (может быть VK / фестивали / ticket-sites)
- «Останови импорт» (нужно уточнить какой именно: VK auto import и т.д.)

## UX / поведение

1. Админ формулирует намерение (например: «покажи события на завтра», «открой очередь VK»).
2. Gemma возвращает один из результатов:
   - **proposals**: 1–3 вариантов действий;
   - **clarify**: уточняющий вопрос (иногда с кнопками вариантов);
   - **unknown**: не удалось однозначно подобрать действие.
   - Для явных латинских/`/slash` подсказок (`rebuild_event 123`, `/recent_imports 48`, `telegraph_cache_stats event`) сначала срабатывает детерминированный роутинг без вызова LLM.
3. Бот показывает:
   - интерпретацию действия,
   - риск‑метку (safe / mutating / dangerous),
   - конкретную команду/действие, которое будет выполнено,
   - кнопки `✅ Подтвердить` / `❌ Отклонить` (+ переключение вариантов, если их несколько).
4. По `✅ Подтвердить` бот **симулирует входящее сообщение** с выбранной командой
   и прогоняет его через текущий `Dispatcher`, поэтому выполняется та же логика,
   что и при ручном вводе команды.

## Safety: почему это не ломает прод и Smart Update

- Ассистент включается **только** по `/assist` или кнопке меню; обычные сообщения
  не переопределяются.
- Любое действие требует явного подтверждения.
- Действия выбираются **только** из явного allowlist в коде (`handlers/admin_assist_cmd.py`).
- Важно: пересланные посты (forward) для Smart Update **не перехватываются** ассистентом:
  если во время ассистента админ переслал пост, ассистент сбрасывается, а сообщение
  обрабатывается обычным forward‑пайплайном.

## Где править / как расширять

- Allowlist и правила валидации: `handlers/admin_assist_cmd.py` (`_allowed_actions()`).
- Базовый промпт: `docs/llm/admin-actions.md` (общие правила; список действий и схема
  дописываются в prompt динамически).
- Команда и кнопка меню: `main.py` (`/assist`, `MENU_ADMIN_ASSIST`).

Особое правило маршрутизации: если пользователь просит **список событий**,
созданных/обновлённых из `Telegram` / `VK` / `/parse` за rolling-window
(`за сутки`, `за 24 часа`, `свежие`, `что создалось`), ассистент должен
предлагать `/recent_imports`, а не уводить в календарный `/events`.

Текущее покрытие `/a`: allowlist синхронизирован с зарегистрированными slash-командами
бота, включая служебные команды вроде `/rebuild_event`, `/telegraph_cache_stats`,
`/telegraph_cache_sanitize`, `/ik_poster`, `/assist_cancel` и stateful `/cancel`.
Если появляется новая команда, тест `tests/test_admin_assist_cmd.py` должен падать,
пока её не добавят в allowlist ассистента.

## Конфигурация

- `ADMIN_ASSISTANT_MODEL` — модель для Gemma/Gemini через Google AI (по умолчанию `gemma-3-27b`).

Правило сопровождения: при добавлении новых команд/функций, которые админ должен
вызывать «простыми словами», добавьте их в allowlist ассистента и обновите
`docs/operations/commands.md` (канонический список команд).
