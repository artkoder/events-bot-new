# Недавние импорты (`/recent_imports`)

> **Команда:** `/recent_imports [hours]`  
> **Доступ:** `superadmin`  
> **Окно по умолчанию:** последние `24` часа

## Назначение

Команда показывает **список событий**, которые были **созданы** или **обновлены**
из внешних пайплайнов за rolling-window по времени:

- Telegram Monitoring (`source_type='telegram'`)
- VK (`source_type LIKE 'vk%'`)
- `/parse` (`source_type LIKE 'parser:%'`)

Это отдельный отчётный слой между:

- `/events` — календарный список событий на конкретную дату;
- `/general_stats` — агрегированные операционные метрики за прошлые 24 часа;
- `/popular_posts` — engagement-отчёт по постам TG/VK, а не список свежих импортов.

## Поведение

- окно считается как `now - hours .. now`;
- если `hours` не указан, используется `24`;
- строки дедуплицируются по `event_id`;
- статус строки:
  - `✅` — событие создано внутри окна (`event.added_at`);
  - `🔄` — событие существовало раньше, но внутри окна получило новый релевантный источник (`event_source.imported_at`);
- в строке показываются:
  - `id` события;
  - компактный статус (`✅` / `🔄`);
  - заголовок c Telegraph-ссылкой, если она есть;
  - дата/время события;
  - агрегированные источники (`Telegram`, `VK`, `/parse`);
  - время последнего релевантного импорта.

Сортировка: по последнему релевантному `imported_at` по убыванию.

## Данные

Источники отчёта:

- `event`
- `event_source`

Фильтр строится по `event_source.imported_at` в окне и по типу источника.

## `/a` и естественные формулировки

Админ-ассистент (`/assist`, `/a`) должен вести сюда запросы вида:

- «список событий из телеграм мониторинга и вк за сутки»
- «что создалось из автоимпорта за 24 часа»
- «покажи свежие импортированные события»

Если пользователь просит **список конкретных событий** по источнику и rolling-window,
это `recent_imports`, а не `/events` и не `/general_stats`.
