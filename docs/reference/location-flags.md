# Location Flags (настройки площадок)

Этот файл задаёт дополнительные “флаги поведения” для некоторых площадок.

Важно: список стандартных имён площадок живёт в `docs/reference/locations.md` и используется для нормализации `location_name`.  
В этом файле строки `location_name` должны **точно совпадать** с каноническим названием площадки.

## allow_parallel_events

По умолчанию (если площадка не перечислена ниже): `allow_parallel_events=false`.

Если `allow_parallel_events=true`, то площадка может проводить **несколько разных событий одновременно** (в один `date/time`) и матчинг/мердж должен быть более осторожным (см. `docs/features/smart-event-update/README.md`).

Важно: это **не про выставки** и другие длинные события с периодом (`date..end_date`). Для них пересечение периодов само по себе не означает дубль — в одном музее/галерее может идти несколько выставок параллельно.

```yaml
locations:
  # Калининградская областная научная библиотека (алиасы: "Научная библиотека",
  # "Калининградская областная научная библиотека") может проводить
  # несколько мероприятий в одно время.
  - location_name: "Научная библиотека, Мира 9, Калининград"
    allow_parallel_events: true

  # Добавляйте сюда другие площадки, где реально бывают параллельные события.
  # - location_name: "..."
  #   allow_parallel_events: true
```
